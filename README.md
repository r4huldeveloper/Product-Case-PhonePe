# PhonePe Mobile Recharge Optimization: Product Case Study

## 📌 Project Overview
This project focuses on streamlining the mobile recharge experience for **PhonePe's 500M+ users**. By identifying friction in the existing 9-step journey, I redesigned the flow into a personalized **6-tap experience**.

### 🔴 The Problem
* **High Cognitive Load:** Users must browse through 40+ generic plans because habitual packs aren't prioritized.
* **Discovery Gap:** The UI promotes global trends over user history, stretching a routine task to 30+ seconds.
* **Redundant Steps:** Repeat users are forced through an unnecessary "Plan Details" confirmation layer.

---

## 🛠 Proposed Solution & Features
We architected three key feature updates to reduce "Time-to-Task" completion:

* **Feature A: Duration-Based Filters** – Replaced marketing themes with intuitive utility filters (28/56/84 days).
* **Feature B: Quick Repeat Shortcut** – Surfaced the "Last Recharged Plan" as the primary CTA directly on the recent contacts list.
* **Feature C: Recommended Tab** – Introduced a smart tab pinning the user's past two recharge details and "Value for Money" options.

---

## 👥 Target Personas
* **The Power User:** Recharges for multiple family members; needs one-tap "Quick Repeat" shortcuts.
* **The Value Seeker:** Habitual recharger; needs a "Recommended" tab for history and value-for-money options.
* **The Budget-Conscious:** Needs duration-based filters to compare utility quickly.

---

## 📈 Success Metrics
* **Primary Metric: Recharge Velocity** – Measuring the % of recharges completed via personalized cards vs. manual search.
* **Secondary Metric: Step Drop-off Rate** – Tracking churn reduction between plan selection and payment.
* **Guardrail Metric: ARPU** – Ensuring increased speed does not cannibalize revenue from high-margin bundled packs.
